# Guia de Configuração, Deploy e Execução do Projeto

Este guia contém as instruções passo a passo para colocar a infraestrutura na nuvem (Vercel e Supabase) em funcionamento e rodar o motor de apuração na sua máquina local.

---

## 1. Configuração do Banco de Dados (Supabase)

Para garantir que o webhook salve o tíquete na nuvem e o motor local consiga resgatá-lo de forma segura, siga os passos abaixo no Supabase:

1. Acesse o painel do [Supabase](https://supabase.com) e crie um projeto gratuito.
2. No menu lateral, acesse **SQL Editor** e clique em **New Query**.
3. Copie o script SQL abaixo, cole no editor e clique em **Run**:

```sql
-- Criar a tabela para armazenamento temporario dos tiquetes
CREATE TABLE tiquetes_cbs (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    cnpj VARCHAR(14) NOT NULL UNIQUE,
    tiquete VARCHAR(100) NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL
);

-- Habilitar Row Level Security (RLS) para seguranca de dados
ALTER TABLE tiquetes_cbs ENABLE ROW LEVEL SECURITY;

-- Criar politica para permitir todas as acoes usando a chave Service Role (chave interna do servidor)
CREATE POLICY "Permitir acesso total via service_role" 
ON tiquetes_cbs 
USING (true) 
WITH CHECK (true);
```

4. Vá em **Project Settings** > **API** e copie os seguintes dados:
   * **Project URL**: Será a sua `SUPABASE_URL`.
   * **service_role API Key** (revelada sob proteção): Será a sua `SUPABASE_SERVICE_ROLE_KEY`.

## 2. Deploy do Webhook na Nuvem (Vercel)

Você pode publicar a sua API de Webhook na Vercel em poucos segundos através do terminal:

### Instalação da Vercel CLI (caso não possua)
Abra o seu terminal na pasta do projeto `cloud-webhook` e execute:
```bash
npm install -g vercel
```

### Deploy do Projeto
1. Certifique-se de estar na pasta `c:\Users\edilson.rodrigues\Documents\26 - Apuração assitida\cloud-webhook`.
2. Execute o comando de login no terminal:
   ```bash
   vercel login
   ```
3. Execute o comando para iniciar o deploy e responda às perguntas padrão (pressione Enter para a maioria):
   ```bash
   vercel
   ```
4. Ao final do deploy, a Vercel gerará um link público (ex: `https://cloud-webhook-receita-xxxx.vercel.app`). **Guarde esta URL base**.

### Configurando as Variáveis de Ambiente no Painel Vercel
No dashboard da Vercel, acesse o seu projeto criado, vá em **Settings** > **Environment Variables** e adicione as seguintes variáveis:

*   `WEBHOOK_SECRET`: Crie uma chave secreta e aleatória de segurança (ex: `SenhaSeguraDoMeuWebhook123!`). Ela protegerá seus endpoints.
*   `SUPABASE_URL`: A URL do seu projeto Supabase.
*   `SUPABASE_SERVICE_ROLE_KEY`: A chave service_role do Supabase obtida no Passo 1.

> [!NOTE]
> Após salvar as variáveis no painel da Vercel, faça um novo deploy para que as alterações façam efeito executando:
> ```bash
> vercel --prod
> ```

## 3. Configuração e Execução do Motor Local (`local-engine`)

Com o webhook publicado e configurado na nuvem, configure o motor local que rodará em sua máquina.

### Preparação do Arquivo `.env`
Acesse a pasta `c:\Users\edilson.rodrigues\Documents\26 - Apuração assitida\local-engine`, abra o arquivo `.env` e preencha com as suas chaves reais:

```env
# Credenciais da API da Receita Federal
RF_CLIENT_ID=SEU_CLIENT_ID_REAL_AQUI
RF_CLIENT_SECRET=SEU_CLIENT_SECRET_REAL_AQUI
RF_BASE_URL=https://api.receitafederal.gov.br

# URL gerada no deploy do seu Webhook na Vercel (Passo 2)
# Exemplo: https://cloud-webhook-receita-xxxx.vercel.app/api/webhook-receita
RF_WEBHOOK_URL=https://sua-api.vercel.app/api/webhook-receita

# Configurações de comunicação do Motor Local com o Webhook da Vercel
# O WEBHOOK_SECRET deve ser exatamente IGUAL ao cadastrado no painel da Vercel
WEBHOOK_SECRET=SenhaSeguraDoMeuWebhook123!
VERCEL_API_URL=https://sua-api.vercel.app

# CNPJ para consulta de debitos (Apenas os 8 primeiros digitos numericos)
CNPJ_APURACAO=12345678
```

### Instalação de Dependências e Inicialização do Motor Local
Abra o seu terminal na pasta do motor local `local-engine` e execute os comandos:

```bash
# 1. Instalar as dependencias
npm install

# 2. Executar o motor local em modo de desenvolvimento
npm run dev
```

O motor iniciará o fluxo completo exibindo no console:
1. Validação se o CNPJ já atingiu o limite de 2 chamadas/dia civil no arquivo local `limites_disparo.json`.
2. Obtenção do Token OAuth2 na API de Produção da Receita Federal.
3. Disparo do POST solicitando a apuração do CNPJ para a Receita.
4. Polling seguro a cada 30 segundos no seu Webhook na Vercel buscando o tíquete.
5. Download automático do JSON completo contendo os débitos de CBS, salvando-o na pasta `local-engine/downloads`.
6. Higienização reativa (remoção automática) do tíquete no Supabase por conformidade e privacidade.
